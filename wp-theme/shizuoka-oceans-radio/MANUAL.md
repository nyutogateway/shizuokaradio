# SHIZUOKA OCEANS RADIO テーマ 運用マニュアル

WordPress テーマ `shizuoka-oceans-radio` の管理・運用手順書。

---

## 0. 【最重要】テーマ更新後に必ずやること

**設定 → パーマリンク設定 → 「変更を保存」を押す**（何も変更せず押すだけでOK）。

今回 CPT の `has_archive` を変更したため、リライトルールの再生成が必要です。**これをやらないと `/program/` `/personality/` が固定ページではなく古いアーカイブのままになります。** 症状が直らない場合はまずこれを疑ってください。

---

## 1. ページ構成とURL設計

### 1-1. 一覧ページは「固定ページ」に一本化

番組・パーソナリティーの一覧は、CPTのアーカイブではなく**固定ページ**で作ります。

| ページ | 必要なスラッグ | 中身の作り方 |
|---|---|---|
| お知らせ | `news` | 設定→表示設定で「投稿ページ」に指定（後述） |
| 番組一覧 | `program` | 固定ページ |
| パーソナリティー | `personality` | 固定ページ |
| リクエスト | `request` | 固定ページ |
| 会社概要 | `company` | 固定ページ＋テンプレート「会社概要（左メニュー付き）」 |
| プライバシーポリシー | `privacypolicy` | 固定ページ |

**スラッグは上の表と厳密に一致させてください。** テーマがこの名前でページを探します。日本語スラッグ（例：`会社概要`）のままだとリンクが繋がりません。

### 1-2. なぜアーカイブを使わないのか

CPT `program` / `personality` は rewrite slug が同名のため、`has_archive => true` だと `/program/` を**アーカイブが先取り**し、同じスラッグの固定ページに永久に到達できませんでした。これが「ナビが全部アーカイブに飛ばされる」の原因です。

現在は `has_archive => false`（[functions.php:74,88](functions.php#L74)）にして `/program/` を固定ページへ開放しています。個別記事は従来どおり `/program/{記事スラッグ}/` です。

> **番組一覧の中身**：アーカイブを止めたので、番組が自動で並ぶことはありません。一覧を出すには固定ページ側でクエリを回す実装が別途必要です（未実装）。

### 1-3. お知らせ（投稿）の設定

1. 固定ページ「お知らせ」をスラッグ `news` で作成（中身は空でよい）
2. **設定 → 表示設定 → 投稿ページ** で「お知らせ」を選択

テーマはこの指定を最優先で読みます（[functions.php:280](functions.php#L280)）。未指定ならスラッグ `news` を探しにいきます。

## 2. ヘッダーナビの管理

### 2-1. メニューを割り当てる

**外観 → メニュー** で作成し、以下2つの位置に割り当てます。ロゴを挟んで左右に分かれるため、メニューも2つ必要です。

| 位置 | 既定の項目 |
|---|---|
| ヘッダー左（ロゴの左側） | お知らせ / 番組一覧 |
| ヘッダー右（ロゴの右側） | パーソナリティー / リクエスト |

**未割り当ての場合は、テーマ内蔵の既定項目（上表）が自動で出ます。** 順番やリンク先を変えたくなった時だけメニューを作れば十分です。

### 2-2. 英字ラベル（NEWS / PROGRAM 等）の入れ方

ナビは `NEWS` の下に `お知らせ` が出る2段構造です。この英字は**メニュー項目の「説明」欄**に入れます。

1. 外観 → メニュー画面の右上「**表示オプション**」を開く
2. 「**説明**」にチェックを入れる（デフォルトでは隠れています）
3. 各メニュー項目を開き、説明欄に `NEWS` `PROGRAM` などを入力

説明欄が空なら英字なしで日本語だけ出力されます（レイアウトは崩れません）。

## 3. フッターの管理

**外観 → メニュー**で以下に割り当て。こちらも未割り当てなら既定項目が出ます。

| 位置 | 既定の項目 |
|---|---|
| フッター（主要メニュー） | お知らせ / 番組一覧 / パーソナリティー / リクエスト |
| フッター（小メニュー） | 会社概要 / 広告料金表 / プライバシーポリシー |

「広告料金表」の既定リンク先は `#`（未設定）です。会社概要ページ内のセクションへ飛ばすか、独立ページを作るか未定のため保留にしています。

## 4. 会社概要ページ

### 4-1. 作り方

固定ページを作成し、**ページ属性 → テンプレート → 「会社概要（左メニュー付き）」**を選択。

### 4-2. 左メニューは自動生成

本文中の `<h2>` を自動的に拾って左サイドの「MENU」を組み立てます（[template-company.php](page-templates/template-company.php)）。**メニューを手書きする必要はありません。** セクションを増やしたい時は H2 で見出しを作るだけです。

### 4-3. 地図の埋め込み

`<iframe>` を「カスタムHTML」ブロックで貼れば、幅100%・16:9で自動表示されます。

> **注意**：`<iframe>` を保存できるのは `unfiltered_html` 権限を持つユーザー（＝シングルサイトの管理者）だけです。**編集者権限だと保存時に地図だけ消えます。**

## 7. トラブルシューティング

### ナビが固定ページではなくアーカイブに飛ぶ / 404になる

1. **設定 → パーマリンク設定 → 保存**（第0章）
2. 固定ページのスラッグが第1章の表と一致しているか確認

### 投稿が全て予約投稿になる

**設定 → 一般 → タイムゾーン**が `UTC+0` 等になっていないか確認。日本時間で入力した時刻をUTCとして解釈されると9時間先の未来と判定され、予約投稿になります。同画面の「現在時刻」が実時刻と一致しているかが判断材料です。タイムゾーンは `UTC+9` ではなく都市名の「**東京**」を選択してください。

### 予約投稿が「予約投稿に失敗しました」になる

WP-Cron が動いていません。WP-Cronはサイトへのアクセス時にしか起動しないため、公開前・低トラフィックだと実行されません。

`wp-config.php` に追加：

```php
define( 'DISABLE_WP_CRON', true );
```

サーバーのcron設定で10分おきに実行：

```
*/10 * * * * /usr/bin/php /home/ユーザー名/ドメイン/public_html/wp-cron.php >/dev/null 2>&1
```

Basic認証やIP制限をかけている場合はループバックリクエストが弾かれるため、`define( 'ALTERNATE_WP_CRON', true );` を検討してください。
